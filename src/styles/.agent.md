# src/styles/

Variants Framer Motion compartilhados. **NÃO é folha de estilo CSS** — pra isso usar Tailwind utility classes ou estender o `@theme {}` em `../App.css`.

## Arquivos

- **`animations.ts`** — variants e configs Framer Motion exportadas pra reuso entre componentes.

### Variants disponíveis

| Nome | Aplicação | Detalhes |
|---|---|---|
| `containerVariants` | Wrapper de listas | `staggerChildren: 0.05` |
| `cardVariants` | Card individual em `motion.div` filho de container | fade + y 14→0, ease-out 0.32s |
| `cardHover` | `whileHover` + `transition` separados | scale 1.005, y -2, spring 380/24 |
| `buttonHover` | `whileHover` + `whileTap` | scale 1.02 hover, 0.98 tap |
| `modalVariants` | Conteúdo do modal | fade + scale 0.96→1, ease-out 0.18s |
| `overlayVariants` | Backdrop do modal | fade-in/out |
| `slideInRight` | Drawer lateral direito | spring 320/32, exit 0.18s |

## Convenções

- **Importar variants daqui**, não recriar inline. Se precisar de variant nova compartilhada, adicionar aqui.
- **Variants só de um componente** podem ficar inline na própria TSX (ex: a animação do `clip-path` do radial reveal no Run/Stop fica no `ProjectCard.tsx`).
- **Não exceder 350ms em duração total** — regra do design system Falcão. Animações lentas viram interrupção.
- **Spring é preferível a tween** pra hovers e transições naturais — cuidar com `stiffness` muito alto (vira jittery) ou muito baixo (vira lento).
- **`prefers-reduced-motion: reduce`** já está coberto globalmente em `App.css` — não precisa repetir checagem por componente.
