# src/

Frontend React + TypeScript. Roda dentro do webview do Tauri (não no browser do sistema).

## Arquivos

- **`main.tsx`** — entrypoint. Cria root React, renderiza `<App />` em StrictMode. Praticamente nunca muda.
- **`App.tsx`** — componente raiz. **Concentra todo o estado da aplicação** (projetos, status, logs, portas, drawer selecionado, modal aberto, busca, toggles). Listeners de eventos Tauri vivem aqui. Atalhos globais (`Esc`, `/`, `Ctrl+K`) registrados num `useEffect`.
- **`App.css`** — única folha de estilo. Inclui:
  - `@import` das 3 fontes (Plus Jakarta Sans, JetBrains Mono, Fraunces)
  - `@import "tailwindcss"`
  - `@theme {}` com tokens canônicos do design system Falcão (`--color-bg-primary`, `--color-text-primary`, `--color-accent-primary` Amber, `--font-display` Fraunces, etc)
  - `:root` com `--noise-texture` (paper-grain SVG inline)
  - `body` com `background-image: var(--noise-texture)` + `background-blend-mode: overlay`
  - `@media (prefers-color-scheme: light)` — light mode automático seguindo tema do sistema
  - `.page-title` (Fraunces, letter-spacing -0.02em)
  - `.radial-reveal` keyframes
  - Custom scrollbar
  - `@media (prefers-reduced-motion: reduce)` — desativa animações pra usuários sensíveis

  **Não criar outros .css** — usar Tailwind utility classes ou estender o `@theme`.
- **`types.ts`** — tipos TS compartilhados que casam com structs Rust serializadas. Manter em sync com `src-tauri/src/process.rs`, `scanner.rs`, `config.rs`.
- **`vite-env.d.ts`** — types do ambiente Vite, gerado.

## Subpastas

- **`assets/`** — imagens importadas via JS (passa pelo bundler).
- **`components/`** — componentes React (cards, drawer, modais, checkbox, system icon).
- **`styles/`** — variants Framer Motion compartilhados (`animations.ts`).

## Padrões importantes

### Estado é centralizado em `App.tsx`

Todo estado vive em `App.tsx` e desce por props. Sem context API, sem zustand, sem nada — o app é pequeno o suficiente. Se isso virar dor, considerar `useReducer` ou um store leve, mas não antecipar.

### Eventos Tauri ↔ React

Backend emite via `app.emit("nome", payload)`. Frontend escuta via `listen<T>("nome", cb)`. Os 4 eventos atuais:

- `log` — `{ id, stream, line }` — uma linha de stdout/stderr de um processo.
- `status` — `{ id, status, code, message }` — mudança de status de um processo (`running`/`stopped`/`crashed`).
- `port` — `{ id, port, url }` — porta detectada nos logs (regex). Filtra a `backend_port` quando configurada.
- `port-allocated` — `{ id, frontend_port?, backend_port? }` — emitido logo após spawn quando há override de porta. UI mostra a porta esperada antes do regex confirmar.

**Nota de serialização:** payloads de eventos Rust→JS chegam em **snake_case** (default do serde). Já os args de comando (JS→Rust) são auto-convertidos camelCase→snake_case pelo Tauri. Cuidado ao adicionar novos.

### Tema (Tailwind v4 + design system Falcão)

Tokens canônicos em `App.css` no `@theme {}` com prefixo `--color-*`. Acessar via:
- Classe direta: `bg-bg-primary`, `text-text-primary`, `bg-accent-primary` (Tailwind v4 cria automaticamente classes a partir do `--color-*`)
- Var CSS direta: `bg-[var(--color-bg-card)]` quando precisar de algo arbitrário
- Fonts: `font-sans` (Plus Jakarta), `font-mono` (JetBrains), `font-display` (Fraunces — ou usar classe `.page-title`)

Cores principais: `bg-primary`, `bg-secondary`, `bg-tertiary`, `bg-card` (glass), `bg-glass`, `text-primary`, `text-secondary`, `text-muted`, `accent-primary` (Amber), `accent-soft`, `success`, `success-soft`, `danger`, `danger-soft`, `border-subtle`, `border-default`. **Não usar paleta default do Tailwind** (`bg-blue-500`, etc) — sempre tokens semânticos.

Light mode é automático via `@media (prefers-color-scheme: light)` — segue GNOME Settings.

### localStorage

Usado pra preferências locais que não fazem sentido no `~/.config/falcao-launcher/config.json`:
- `falcao-launcher.autoOpenBrowser` — toggle "abrir browser auto"
- `falcao-launcher.showHidden` — toggle "mostrar projetos ocultos"

Outras prefs novas seguem o mesmo prefix.

### Motion

Variants compartilhados em `styles/animations.ts`. Padrões:
- Cards: `cardHover` (scale 1.005, y -2, spring 380/24).
- Botões: `whileHover: 1.02, whileTap: 0.96`.
- Modais: `modalVariants` (fade+scale 0.96→1) + `overlayVariants` (fade).
- Drawer: `slideInRight` (spring 320/32).
- Listas: `containerVariants` com `staggerChildren: 0.05`.

**Regra do design system**: feedback colors NUNCA pop instantaneamente — sempre via radial reveal (`clip-path: circle(0% → 150%)`, ~0.45s). Implementado no botão Run/Stop com `AnimatePresence`.
