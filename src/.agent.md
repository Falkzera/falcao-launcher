# src/

Frontend React + TypeScript. Roda dentro do webview do Tauri (não no browser do sistema).

## Arquivos

- **`main.tsx`** — entrypoint. Cria root React, renderiza `<App />` em StrictMode. Praticamente nunca muda.
- **`App.tsx`** — componente raiz. **Concentra todo o estado da aplicação** (projetos, status, logs, portas, drawer selecionado, busca). Listeners de eventos Tauri vivem aqui. Atalhos globais (`Esc`, `/`, `Ctrl+K`) registrados num `useEffect`.
- **`App.css`** — única folha de estilo. Inclui `@import "tailwindcss"` e o bloco `@theme {}` com os tokens do design system. **Não criar outros .css** — usar Tailwind utility classes ou estender o `@theme`.
- **`types.ts`** — tipos TS compartilhados que casam com structs Rust serializadas. Manter em sync com `src-tauri/src/process.rs` e `scanner.rs`.
- **`vite-env.d.ts`** — types do ambiente Vite, gerado.

## Subpastas

- **`assets/`** — imagens importadas via JS (passa pelo bundler).
- **`components/`** — componentes React (cards, drawer).

## Padrões importantes

### Estado é centralizado em `App.tsx`

Todo estado vive em `App.tsx` e desce por props. Sem context API, sem zustand, sem nada — o app é pequeno o suficiente. Se isso virar dor, considerar `useReducer` ou um store leve, mas não antecipar.

### Eventos Tauri ↔ React

Backend emite via `app.emit("nome", payload)`. Frontend escuta via `listen<T>("nome", cb)`. Os 3 eventos atuais:

- `log` — `{ id, stream, line }` — uma linha de stdout/stderr de um processo.
- `status` — `{ id, status, code, message }` — mudança de status de um processo (`running`/`stopped`/`crashed`).
- `port` — `{ id, port, url }` — porta detectada nos logs.

**Nota de serialização:** payloads de eventos Rust→JS chegam em **snake_case** (default do serde). Já os args de comando (JS→Rust) são auto-convertidos camelCase→snake_case pelo Tauri. Cuidado ao adicionar novos.

### Tema (Tailwind v4)

Tokens em `App.css` no `@theme {}`. Acessar via:
- Classe direta: `bg-bg`, `text-muted` (Tailwind v4 cria automaticamente classes a partir do `--color-*`)
- Var CSS direta: `bg-[var(--color-accent)]` quando precisar de algo arbitrário

Cores principais: `bg`, `surface`, `surface-2`, `border`, `text`, `muted`, `accent`, `success`, `danger`.

### localStorage

Usado pra preferência de "abrir browser auto" (`falcao-launcher.autoOpenBrowser`). Outras prefs novas seguem o mesmo prefix.
