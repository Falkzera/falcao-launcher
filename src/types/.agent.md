# types — agent.md

## Propósito
Definições TypeScript de tipos compartilhados pelo frontend. Espelham os tipos serializados pelos commands Tauri (lado Rust) e mantêm o front type-safe sem precisar regenerar bindings.

## Arquivos
- `monitor.ts` — tipos do VM Monitor:
  - Fase 1: `VmStatus`, `ContainerInfo`, `MetricPoint`, `MetricSource`, `MetricBucket`, `WindowKey`, `HealthCheckSummary` (espelham `monitor-shared` + `src-tauri/src/monitor/queries.rs`).
  - Sprint 2 (Vercel stacks): `VercelState`, `VercelDeploymentRow`, `StackSummary`, `StackDetail` (espelham `src-tauri/src/monitor/stacks.rs`). `StackDetail.vercel_history` traz até 10 deploys ordenados do mais recente.

## Padrões
- Cada arquivo agrupa tipos por feature (não criar um único `types.ts` gigante).
- Manter sincronizado com Rust manualmente. Tauri serializa via serde em `snake_case` por padrão (não camelCase) — TS deve casar.
- Não importar de `src/types/` em arquivos Rust — esses tipos são só consumo do front.

## Decisões recentes
- 2026-05-06: pasta criada pra Fase 1 do monitor.
- 2026-05-07 (Sprint 2): adicionados tipos Vercel/Stack. `VercelState` é union string-like (`"READY" | "ERROR" | ... | (string & {})`) — sentinel `string & {}` aceita estados raros não documentados sem perder autocomplete dos comuns.

## Pendências conhecidas
- Sem geração automática (ts-rs ou similar). Drift de schema é responsabilidade humana — vigiado em code review.
