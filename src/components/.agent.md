# src/components/

Componentes React específicos do launcher. **Sem biblioteca de UI** (não usamos shadcn-CLI nem nada) — cada componente é escrito à mão, simples, ~30-200 linhas. Variants Framer Motion compartilhados ficam em `../styles/animations.ts`.

## Arquivos

- **`ProjectCard.tsx`** — card de um projeto no grid:
  - Avatar (favicon real do projeto OU iniciais sobre cor gerada por hash do nome)
  - Nome (Plus Jakarta semibold) + path (JetBrains Mono `text-muted`)
  - Indicador de status (idle/running/crashed) com pulse animado
  - Chip do package manager + script detectado, chip de porta clicável
  - Ícones de ação no hover: olho (esconder), engrenagem (configurar)
  - Sempre visíveis: VSCode, Ghostty (`<SystemIcon>`), Run/Stop
  - **`motion.div` com `cardHover`** (scale 1.005, y -2, spring) + glass (`bg-bg-card` + `backdrop-blur-md`)
  - **Botão Run/Stop tem radial reveal**: `AnimatePresence` com `clip-path: circle(0% → 150%)` quando estado muda — feedback colors NÃO podem pop, sempre via reveal.
  - **Visual de oculto** quando `project.hidden`: `border-dashed`, opacity 60%, saturate 50%, chip "OCULTO" no canto, hover restaura.

- **`LogsDrawer.tsx`** — drawer lateral direito (480px):
  - **`motion.aside` com `slideInRight`** (spring 320/32) — entra/sai animado.
  - Header: bolinha de status, nome do projeto (`.page-title`), chip da porta, botões `clear` e `✕`.
  - Body: lista de linhas com cor diferente pra stderr (vermelho suave).
  - Auto-scroll com sticky bottom: `stickyRef` desliga o scroll automático se o usuário rolar pra cima.
  - Footer: contador de linhas.

- **`ProjectConfigModal.tsx`** — modal de configuração por projeto:
  - **`motion.div` com `modalVariants` + `overlayVariants`**.
  - Inputs: porta frontend + porta backend (mono, 1024-65535).
  - Seção **Logo do projeto**: chama `list_icon_candidates` no Rust, exibe grid de thumbnails clicáveis dos candidatos detectados (4 colunas). Thumbnail ativo tem ring laranja. Botão "voltar ao auto" limpa o override.
  - Empty state quando não há candidatos com mensagem amigável.
  - Salva via `set_project_config` (commando Rust). Chama `onSaved?.()` pra `App.tsx` re-scanear.

- **`AddProjectModal.tsx`** — modal pra adicionar projeto por path arbitrário:
  - Input de texto, autoFocus, Enter pra submeter.
  - Chama `add_extra_path` no Rust (canonicaliza, valida que é dir).
  - `onAdded` re-fetcha a lista no parent.

- **`Checkbox.tsx`** — checkbox custom seguindo padrão Falcão System Design:
  - `bg-bg-tertiary` unchecked, `bg-accent-primary` checked.
  - **Animação assinatura**: ✓ desenhado em `pathLength: 0 → 1` em 0.22s + scale 0.5 → 1 + opacity. `whileTap: 0.88` pra feedback tátil.
  - `<input type="checkbox">` interno é `sr-only` (visualmente escondido).

- **`SystemIcon.tsx`** — wrapper que usa `resolve_icon` do Rust pra pegar ícone do tema XDG ativo:
  - Cache estático (`Map`) por `name@size` — primeira render dispara fetch, próximas usam cache.
  - Fallback pra `<img src={fallback}>` quando o sistema não tem o ícone.
  - Usado pelos botões VSCode/Ghostty no card.

## Convenções

- **Props sempre tipadas com `type Props = { ... }`**, não `interface`. Mais fácil de estender com `&`.
- **`clsx` pra composição de classes condicionais.** Não usar `cn()` (não temos shadcn).
- **Não exportar default.** Usar named exports.
- **Stop propagation em handlers** quando o componente é clicável e tem buttons internos.
- **Não escrever lógica de negócio aqui.** Cards/modais/drawer só renderizam props e disparam callbacks. Estado e `invoke()` ficam no `App.tsx`. Exceções pontuais: botões Run/Stop, abrir VSCode, abrir Ghostty, ocultar/configurar — chamam `invoke` direto porque não impactam state externo.
- **Motion via Framer**: importar variants de `../styles/animations.ts`. NÃO criar variants ad-hoc dentro do componente — extrair pra reuso.
- **Tokens via classe Tailwind**: `bg-bg-secondary`, `text-text-primary`, `border-border-subtle`, `bg-accent-primary`, etc. Quando precisar de algo arbitrário (ex: shadow custom), usar `bg-[var(--color-bg-card)]`.
- **`page-title` pra display**: aplica Fraunces serif + letter-spacing. Usar em headers de modal/drawer e no título principal.
