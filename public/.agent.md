# public/

Assets estáticos servidos pelo Vite na raiz do servidor (`/`). Diferente de `src/assets/` (que passa pelo bundler).

## Quando usar `public/` vs `src/assets/`

| Caso | Pasta |
|---|---|
| Imagem referenciada por path string em runtime | `public/` |
| Imagem importada por código (`import logo from '...'`) | `src/assets/` |
| Robots.txt, manifest.json, favicon do dev server | `public/` |
| Qualquer asset que precisa hash no nome (cache busting) | `src/assets/` |
