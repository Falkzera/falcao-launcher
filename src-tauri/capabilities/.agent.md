# src-tauri/capabilities/

Sistema de permissões do Tauri 2. **Cada plugin precisa estar autorizado** aqui pra ser invocável da janela.

## Arquivos

- **`default.json`** — permissões da janela `main` (a única que temos).

## Permissões atuais

```json
{
  "windows": ["main"],
  "permissions": [
    "core:default",        // permissões básicas do Tauri (event, window, app)
    "opener:default"       // tauri-plugin-opener (abrir URLs/arquivos)
  ]
}
```

## Como adicionar uma nova capability

1. Instalar o plugin: `pnpm add @tauri-apps/plugin-XYZ` no JS-side e adicionar `tauri-plugin-xyz` no `Cargo.toml`
2. Registrar no `lib.rs`: `.plugin(tauri_plugin_xyz::init())`
3. Adicionar `"xyz:default"` (ou permissões mais específicas como `"xyz:allow-read"`) à lista `permissions` aqui
4. Reiniciar `pnpm tauri dev`

## Por que isso importa

Capabilities são a fronteira de segurança. Webview executando código JS arbitrário só pode chamar APIs nativas listadas aqui. **Não adicionar `core:allow-everything` ou similar** — quebra o modelo de segurança do Tauri.
