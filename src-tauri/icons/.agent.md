# src-tauri/icons/

Ícones do bundle nativo. **Gerados, não editados à mão.**

## Como foram gerados

```bash
cd ~/Projects/falcao-launcher
rsvg-convert -w 1024 -h 1024 icon-source.svg -o icon-1024.png
pnpm tauri icon icon-1024.png
```

Isso popula automaticamente todos os tamanhos pros vários sistemas operacionais.

## Conteúdo

### Linux/Desktop

- `32x32.png`, `64x64.png`, `128x128.png`, `128x128@2x.png`, `icon.png` — usados pelo bundle Linux/macOS
- `icon.icns` — formato Apple, usado pelo bundle macOS
- `icon.ico` — formato Windows, usado pelo bundle .exe

### Windows Store / UWP

- `Square*Logo.png` (várias resoluções) e `StoreLogo.png` — pro caso de empacotar pra MS Store. Não usamos, mas o `tauri icon` gera.

### Mobile

- `android/` — ícones adaptativos (`mipmap-*` em várias densidades). Só relevante se um dia rodar `tauri android dev`.
- `ios/` — `AppIcon-*` em várias resoluções pra iOS. Idem.

## Quando regenerar

Sempre que mudar o `../../icon-source.svg`. Os ícones são embedados em build-time no binário Rust via `tauri::generate_context!()` — depois de regenerar, **forçar recompile** com `touch src-tauri/src/lib.rs`.

## Importante

Esses ícones são pro **bundle nativo** (deb/rpm/exe/dmg). Não confundir com:

- `~/.local/share/icons/hicolor/*/apps/falcao-launcher.png` — ícone do app **instalado** no GNOME (manualmente populado).
- `public/favicon.*` — favicon do dev server do Vite (não usamos).
