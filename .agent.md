# falcao-launcher — raiz

Raiz do monorepo de single binary. Frontend (Vite/React) e backend (Rust/Tauri) convivem aqui.

## Arquivos

- **`CLAUDE.md`** — overview do projeto, carregado automaticamente pelo Claude Code. Ler antes de tudo.
- **`package.json`** — deps e scripts JS. Os scripts úteis: `pnpm tauri dev` (sobe app), `pnpm tauri build` (release), `pnpm dev` (só Vite, sem Tauri — raramente útil).
- **`pnpm-lock.yaml`** — lockfile do pnpm. Não editar à mão.
- **`vite.config.ts`** — config do Vite. Plugin `@tailwindcss/vite` registrado, porta fixa `1420` (Tauri exige), `src-tauri/**` ignorado pelo watcher.
- **`tsconfig.json`** + **`tsconfig.node.json`** — config TS. `tsconfig.node.json` é o do Vite (Node-side); o principal é `tsconfig.json`.
- **`index.html`** — entrypoint HTML. Vite injeta o bundle do React aqui.
- **`README.md`** — boilerplate do `create-tauri-app`. Pode ignorar.
- **`icon-source.svg`** — fonte do ícone do app (gradient laranja + F preto + play triangle). Editar aqui se quiser mudar o ícone.
- **`icon-1024.png`** — render PNG do SVG, entrada de `pnpm tauri icon`. Regenerar com `rsvg-convert -w 1024 -h 1024 icon-source.svg -o icon-1024.png`.

## Subpastas

- **`src/`** — código frontend (React).
- **`src-tauri/`** — código backend (Rust) + config nativa.
- **`public/`** — assets estáticos servidos pelo Vite.
- **`.claude/`** — config do Claude Code (settings.json com permissões).
- **`node_modules/`** — gerado, não tocar.

## Mudou ícone? Como propagar

```bash
rsvg-convert -w 1024 -h 1024 icon-source.svg -o icon-1024.png
pnpm tauri icon icon-1024.png
touch src-tauri/src/lib.rs   # força recompile (ícones embedam em build-time)
```

Pra atualizar também o ícone do app instalado no GNOME:

```bash
for size in 32 128 256 512; do
  rsvg-convert -w $size -h $size icon-source.svg -o ~/.local/share/icons/hicolor/${size}x${size}/apps/falcao-launcher.png
done
gtk-update-icon-cache -t ~/.local/share/icons/hicolor
```
